Figure 1
The system as one closed loop
The whole system at a glance. A one-time generation pipeline (gray) turns a skill spec into a layout genome that is rendered, compiled, and shown. The live window then emits a routing ledger that drives the evolution loop (jade), which writes a new genome (closing the loop) and accrues confirmed moves into a pattern library that seeds future generation.

A genome and the interface it renders
layout genome (abridged JSON)
{
  "tree": { "split-h", sizes [32, 68]:

        { "split-v", sizes [66, 34]:

  "slots": {
    chat:     AgentChat      role: main
    diff:     DiffViewer     mode: unified
    findings: FindingsPanel  role: log },

  "routing": [
    payloadClass diff        → diff
    event tool_execution_*  → findings
    payloadClass text        → chat
    fallback                → chat ] }
rendered layout (split-h 32 / 68)
AgentChatrole: main slot: chat
DiffViewermode: unified slot: diff
FindingsPanelrole: log slot: findings
data plane (routing) payloadClass:diff→diff event:tool_execution_*→findings payloadClass:text→chat fallback→chat
The interface is a function of the genome. Slot ids (color-coded) connect the JSON definition to the rendered panels; the routing table is the data plane that decides where each incoming agent event is shown. Mutation operates on this document, and telemetry attaches to the stable slot ids.

Adaptation you can see. The resize morph fires when a slot carries a dominant share of routed output but holds a minority of the space. It is computed entirely from the routing ledger, is a single reversible edit, and ships under a hypothesis with automatic rollback if the change does not hold.
